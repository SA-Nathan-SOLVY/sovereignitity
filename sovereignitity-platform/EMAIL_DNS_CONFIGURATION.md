## üìß DNS Records Configuration

### Required DNS Records

#### 1. A Record (Mail Server)
```
Type:     A
Name:     mail.solvy.chain
Value:    [YOUR_SERVER_IP]
TTL:      3600 (or Auto)
```

**Purpose:** Points the mail subdomain to your Mailcow server IP address.

**Action Required:** Replace `[YOUR_SERVER_IP]` with your actual Mailcow server IP address.

---

#### 2. MX Record (Mail Exchange)
```
Type:     MX
Name:     solvy.chain
Priority: 10
Value:    mail.solvy.chain
TTL:      3600 (or Auto)
```

**Purpose:** Tells other mail servers where to deliver emails for @solvy.chain addresses.

**Priority:** 10 (lower numbers = higher priority)

---

#### 3. SPF Record (Sender Policy Framework)
```
Type:     TXT
Name:     solvy.chain
Value:    "v=spf1 mx ~all"
TTL:      3600 (or Auto)
```

**Purpose:** Prevents email spoofing by specifying which servers can send email on behalf of your domain.

**Breakdown:**
- `v=spf1` - SPF version 1
- `mx` - Allow servers listed in MX records to send email
- `~all` - Soft fail for all other servers (marks as spam but doesn't reject)

**Alternative (Stricter):**
```
"v=spf1 mx -all"
```
Use `-all` for hard fail (rejects unauthorized senders)

---

#### 4. DMARC Record (Domain-based Message Authentication)
```
Type:     TXT
Name:     _dmarc.solvy.chain
Value:    "v=DMARC1; p=none; rua=mailto:admin@solvy.chain"
TTL:      3600 (or Auto)
```

**Purpose:** Provides instructions to receiving mail servers on how to handle emails that fail SPF/DKIM checks.

**Breakdown:**
- `v=DMARC1` - DMARC version 1
- `p=none` - Policy: Monitor only (don't reject)
- `rua=mailto:admin@solvy.chain` - Send aggregate reports to this address

**Policy Options:**
- `p=none` - Monitor mode (recommended for initial setup)
- `p=quarantine` - Send suspicious emails to spam
- `p=reject` - Reject emails that fail authentication

## üîê Additional Recommended Records

### 5. DKIM Record (DomainKeys Identified Mail)
```
Type:     TXT
Name:     dkim._domainkey.solvy.chain
Value:    [Generated by Mailcow]
TTL:      3600 (or Auto)
```

**Purpose:** Cryptographically signs outgoing emails to verify they haven't been tampered with.

**Note:** Mailcow will generate this record automatically. You'll need to copy it from Mailcow admin panel and add it to your DNS.

**To Get DKIM Record from Mailcow:**
1. Login to Mailcow admin panel
2. Go to Configuration ‚Üí Configuration & Details
3. Select your domain (solvy.chain)
4. Click "DKIM" tab
5. Copy the public key
6. Add as TXT record in DNS

---

### 6. Reverse DNS (PTR Record)
```
Type:     PTR
IP:       [YOUR_SERVER_IP]
Value:    mail.solvy.chain
```

**Purpose:** Maps your server IP back to your mail server hostname. Critical for email deliverability.

**Note:** This must be set by your hosting provider or VPS provider, not in your domain DNS.

**How to Set:**
- Contact your hosting provider (AWS, DigitalOcean, Linode, etc.)
- Request PTR record for your server IP
- Provide: `mail.solvy.chain`

## üö® Common Issues & Solutions

### Issue 1: Emails Going to Spam
**Causes:**
- Missing or incorrect SPF record
- No DKIM signature
- No reverse DNS (PTR)
- Low sender reputation (new server)

**Solutions:**
- Verify all DNS records are correct
- Enable DKIM in Mailcow
- Set up PTR record with hosting provider
- Warm up IP by sending gradually increasing volume

---

### Issue 2: Cannot Receive Emails
**Causes:**
- MX record not set or incorrect
- Firewall blocking port 25
- Server IP not pointing correctly

**Solutions:**
- Verify MX record: `dig MX solvy.chain`
- Check firewall: `sudo ufw status`
- Ensure ports 25, 465, 587, 993, 995 are open

---

### Issue 3: DKIM Not Working
**Causes:**
- DKIM record not added to DNS
- DKIM record syntax error
- DKIM not enabled in Mailcow

**Solutions:**
- Copy DKIM key exactly from Mailcow
- Remove spaces and line breaks
- Verify with: `dig TXT dkim._domainkey.solvy.chain`
